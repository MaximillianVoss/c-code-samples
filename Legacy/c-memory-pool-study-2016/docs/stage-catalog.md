# Каталог исторических этапов

## 1. `MemoryManager`

Дата source-tip: `2016-11-09T16:25:37Z`.

Первый файл `Main.c` похож на набросок allocator из K&R со смещениями в
единицах `Header`. Он не компилируется: отсутствует `getMemBlock`, тип
`Header` используется без `struct`, не подключен `bool`, а объявление
`static public` не является C. Начальный заголовок пула не инициализируется;
удаление головы и слияние у границы списка также некорректны.

## 2. `MemoryManagerC++`

Дата source-tip: `2016-11-09T19:04:38Z`.

Вторая попытка использует статический 150-байтовый буфер. Смещения в байтах
прибавляются к `chunk_header *`, поэтому адрес умножается на размер заголовка.
Та же ошибка есть при вычислении payload и при освобождении. Реализация не
проверяет double free, не объединяет соседние блоки и использует
непереносимые форматы `printf` для `size_t` и указателей.

## 3. `MemoryManagerRC`

Дата source-tip: `2016-12-19T23:46:54Z`.

Последний вариант выделяет API в `memallocator.h`, но `meminit` и успешная
ветка `memalloc` не возвращают значения. Конец пула вычисляется за пределами
переданного буфера; заголовок блока смешивается с payload-указателем;
обход списка может разыменовать `NULL`. Проверка утечек учитывает только
размер головного блока и не доказывает целостность allocator.

Файл `code.txt` сохраняет промежуточную копию RC-реализации. Комментарии в
`memallocator.c` оставлены в исходной кодировке и не перекодировались.
